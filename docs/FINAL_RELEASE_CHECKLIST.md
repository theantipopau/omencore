### 4.3 Contradiction identified — `CHANGELOG.md` vs `CHANGELOG_v3.3.1.md` on STEP-03

These two documents describe the outcome of STEP-03 (locale-safe exception filter) differently:

| Document | Statement |
|---|---|
| `CHANGELOG.md` line 32 | *"The locale-specific exception suppressor in `OnUnobservedTaskException` is **retained** as a defence-in-depth safety net and **extended** to use a stack-trace check…"* |
| `CHANGELOG_v3.3.1.md` line 64 | *"The English-only exception suppressor (`ex.Message.Contains("different thread")`) … has been **removed**. Two locale-safe checks are **retained** as the safety net…"* |

**Verified ground truth** (inspected `App.xaml.cs` lines 1117–1175):  
- `ex.Message.Contains("different thread")` is **not present** in the current source.  
- The filter uses `ex.StackTrace?.Contains("System.Windows.Threading.Dispatcher", ...)` and `ex.TargetSite?.DeclaringType?.FullName?.StartsWith("System.Windows.", ...)`.  
- A code comment at line ~1155 explicitly states: *"The English string match ("different thread") was removed in STEP-03"*.

**Conclusion:** `CHANGELOG_v3.3.1.md` is correct. `CHANGELOG.md` contains a stale description that says the suppressor was "retained and extended" — it was in fact **replaced** (the English string match removed, two locale-safe checks substituted). The `CHANGELOG.md` entry was written before STEP-03 was applied and was not updated.

> **Action required — `CHANGELOG.md` STEP-03 description:** The `[3.3.1]` `### Fixed` first bullet in `CHANGELOG.md` should be updated so its final sentence accurately says the English string match **was removed** and the locale-safe checks were **added as its replacement**, rather than saying the suppressor was "retained … and extended". This is a documentation correctness issue; the code is correct.
