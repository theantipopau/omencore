## Version Resolution Flow (Post-Update)

```
VERSION.txt         → "3.3.1"       ← primary source (AppVersionProvider, AutoUpdateService)
OmenCoreApp.csproj  → 3.3.1.0       ← assembly fallback (FileVersion attribute)
InformationalVersion → "3.3.1"      ← NuGet / about dialog fallback
installer .iss       → "3.3.1"      ← installer product version + output filename
Avalonia _appVersion → "3.3.1"      ← pre-init XAML designer fallback
artifacts/version.json → "3.3.1"    ← release manifest (SHA256 pending Linux build)
```

All six sources are now consistent. The app will report `3.3.1` in:
- About window
- Settings page version label
- Main window version badge
- Diagnostic export bundles
- Tray tooltip
- Auto-update "current version" comparison
- Linux CLI `--version` output
