## Executive Summary

v3.6.0 is a significant lightweight resource optimization release focused on reducing CPU/RAM usage when idle, minimized, or in tray-only mode without compromising active fan control, RGB, tuning, or diagnostics reliability.

### Key Achievements

**GitHub Issue #125 Resolution** ✅  
- **Issue**: "Fans either go max or zero"
- **Root Cause**: Custom fan curves could report successful write but leave RPM at 0
- **Solution**: Detect zero-RPM-with-duty condition after 12 seconds and send bounded wake pulse (35-60%, 60s cooldown)
- **User Impact**: No more manual mode switching needed to recover from stuck fans

**Startup Performance** ✅  
- Removed forced Dashboard/SystemControl lazy-loads from tray startup
- Removed RGB provider (Corsair/Logitech/Razer/OpenRGB) initialization from startup
- Deferred conflict monitoring (Afterburner/RTSS/XTU) until those tabs are opened
- **Result**: Cold startup is now lighter with deferred non-essential initialization

**Monitoring Efficiency** ✅  
- Added adaptive static-tray sampling for low-overhead sessions
- Tray-only idle can now reach 10s ultra-low cadence (from 5s)
- Memory Optimizer telemetry pauses when tab is hidden
- **Result**: Significantly reduced background polling when window is minimized/tray-only

**Fan & Performance Reliability** ✅  
- Fixed Auto preset handoff after Max/policy transitions (no GPU power collapse)
- Rebalanced built-in Auto/Gaming/Extreme curves (70-80C no longer ramps to Max)
- Improved WMI `100%` fan requests to use protocol ceiling (not hardcoded 55 level)
- Added explicit Extended GPU power mode support (RTX 4080/5080)
- **Result**: Thermal behavior is more predictable and curve transitions are smoother

**Resource Diagnostics** ✅  
- Added `resource-footprint.txt` to diagnostics export
- Captures OmenCore/hardware-worker process footprint, active timers, cadence reason
- Live cadence card in Settings shows tier, reason, and blockers
- **Result**: Users can see actual resource usage and optimization state

## Deviations & Notes

### Known Limitations
- Full solution build with `--no-restore` fails on Linux project until `net8.0/win-x64` asset is resolved
- DiagnosticExportSnapshotTests validation pending due to sandbox profile limit gate
- Physical hardware validation requires test systems (in progress by hardware partners)

### Design Decisions
- Zero-RPM wake pulse uses 35-60% bounded range (avoids loud 100%) with 60s cooldown (prevents spam)
- Tray-only ultra-low (10s) only when no fan/OSD blockers active (safety-first)
- Legacy polling-interval config normalized on save but not exposed in UI (backward compatibility)
- Auto preset GPU power preserved via curve-payload detection (more intuitive transitions)

### Backward Compatibility
- Config migration: Existing polling-profile settings read but ignored
- Legacy fan presets continue to work
- Game profiles and automation rules preserved
