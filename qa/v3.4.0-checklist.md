# OmenCore v3.4.0 Release QA Checklist

Use this checklist to validate release readiness for the v3.4.0 cut.

## 1) Preflight

- [ ] `dotnet build OmenCore.sln -c Release` succeeds with no errors
- [ ] `dotnet test OmenCore.sln` passes
- [ ] `VERSION.txt` matches target release version
- [ ] `CHANGELOG.md` and `docs/CHANGELOG_v3.4.0.md` are aligned

## 2) Windows Core Flows

- [ ] Startup as Administrator succeeds without error dialogs
- [ ] Dashboard renders live telemetry (CPU/GPU temp, fan RPM, power)
- [ ] Fan profile cards are visible and selectable (Max/Extreme/Gaming/Auto/Silent/Custom)
- [ ] Custom fan curve apply does not cap CPU package power to incorrect low value
- [ ] Performance mode switching remains responsive (no extended UI stalls)
- [ ] Keyboard lighting apply works and persists for supported models
- [ ] Print Screen key opens Snipping Tool while OmenCore is running

## 3) Monitoring Resilience

- [ ] Suspend/resume recovers telemetry within expected grace window
- [ ] Worker interruption triggers automatic recovery path
- [ ] Telemetry stale/degraded states surface clear UI messaging

## 4) Model Identity and Capability

- [ ] Model Identity Resolution card reports expected source/confidence
- [ ] Capability gating hides unsupported controls on constrained models
- [ ] Added/updated product IDs resolve to expected capability profile

## 5) Linux Packaging and Runtime

- [ ] `build-linux-package.ps1` completes successfully
- [ ] Produced Linux zip includes CLI + GUI payload and checksum output
- [ ] `omencore-cli status` runs successfully on a Linux test machine
- [ ] GUI startup works or documents software-render fallback path clearly

## 6) Windows Packaging

- [ ] `build-installer.ps1` completes successfully
- [ ] Installer launches, installs, and starts app correctly
- [ ] Portable zip launches correctly on clean test environment
- [ ] Published checksums are present and match artifacts

## 7) Updater Safety

- [ ] Update check succeeds against GitHub release endpoint
- [ ] Download requires SHA256 and rejects missing-hash releases
- [ ] Update download uses staged file and final artifact verifies hash
- [ ] Stale files in `%TEMP%\OmenCore\Updates` are pruned as expected

## 8) Scope Freeze (v3.4.0 policy)

- [ ] Bloatware Manager receives bug fixes only (no new features)
- [ ] Memory Optimizer receives bug fixes only (no new features)
- [ ] System Optimizer receives bug fixes only (no new features)
- [ ] Game Library receives bug fixes only (no new features)

## 9) Release Notes and Docs

- [ ] README release/version references are current
- [ ] INSTALL docs and Linux guide are consistent with shipped behavior
- [ ] Antivirus FAQ links and wording are up to date

## 10) Final Sign-off

- [ ] No open critical blockers remain in `docs/roadmap-3.4.0.md`
- [ ] Release artifacts verified and uploaded
- [ ] Tag/release notes prepared
- [ ] Final smoke test on at least one OMEN and one Victus profile
